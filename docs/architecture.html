Complete implementation view for engineers and IT reviewers, covering architecture, data, security, test strategy, and deployability.

Runtime and Application Architecture

  • Frontend: React Native web-capable client with domain screens (auth, catalogue, artists, collections, cart, checkout, contact, admin).
  • Backend: Express application split into app bootstrap, route modules, controller handlers, service layer utilities, and repository-backed data access.
  • Data: SQLite persistence via repository modules for users, catalogue entities, cart, orders, reset tokens, contacts, and admin audit records.
  • Flow: client service modules call REST endpoints, middleware enforces auth and request constraints, and repositories isolate SQL concerns.

Architecture Design Principles

Client React Native UI

Screens, navigation, auth context, and cart context capture interactions and domain state.

→
Service Boundary Express API

Routes, controllers, services, and middleware enforce business rules, auth, and request handling.

→
Persistence Repository + SQLite

Repositories isolate SQL access for catalogue, orders, contacts, users, reset tokens, and admin data.

The visual design shows the core request path: client action, API execution, and repository-backed persistence.

  • Separation of concerns: routing, business logic, and persistence are intentionally isolated.
  • Feature traceability: screens map to service calls, service calls map to API endpoints, and API endpoints map to repositories.
  • Operational clarity: startup checks, monitoring hooks, and configuration validation are treated as part of the architecture design.

Feature Specifications

  • Authentication: register/login, token-based session handling, secure storage utilities, reset-email and token flow support.
  • Catalogue and discovery: artists, collections, exhibitions, artworks, and shop products with clickable detail interactions.
  • Commerce: cart operations, checkout orchestration, payment provider abstraction, and status visibility.
  • Admin and operations: role-aware access, dashboard-oriented features, and contact/message management pathways.

Security, Reliability, and Delivery Specs

  • Security controls: auth middleware, route protection, rate-limiting support, and controlled password reset lifecycle.
  • Config safety: startup and config-check services with script-based validation to catch environment issues early.
  • Test coverage: backend Jest suites for auth/cart/catalogue/checkout/admin/startup/contact plus frontend Jest suites for screen behavior.
  • CI/CD: automated workflows for test execution, security checks, and publication tasks; demo health checks for seeded data thresholds.
